#### [#1] createServiceClient() 쿠키 제거
**파일:** `src/lib/supabase/server.ts`

**문제:** SERVICE_ROLE 클라이언트가 사용자 세션 쿠키를 주입받고 있었음. SERVICE_ROLE은 RLS를 전면 우회하므로 쿠키가 무의미할 뿐 아니라, 세션 덮어쓰기 위험이 있었음.

**수정 전:**
```typescript
export function createServiceClient() {
    const cookieStore = cookies()
    return createServerClient(
        process.env.NEXT_PUBLIC_SUPABASE_URL!,
        process.env.SUPABASE_SERVICE_ROLE_KEY!,
        { cookies: { get, set, remove } } // ← 쿠키 주입
    )
}
```

**수정 후:**
```typescript
import { createClient as createSupabaseClient } from '@supabase/supabase-js'

export function createServiceClient() {
    return createSupabaseClient(
        process.env.NEXT_PUBLIC_SUPABASE_URL!,
        process.env.SUPABASE_SERVICE_ROLE_KEY!,
        { auth: { persistSession: false, autoRefreshToken: false } }

}
```

**검증:** 쿠키 의존성 제거, 세션 없이 동작 확인. 기존 서비스 클라이언트 호출부 영향 없음 (SERVICE_ROLE은 원래 세션 불필요).

#### [#2] hashIp() — SHA-256으로 교체
**파일:** `src/lib/utils.ts`

**문제:** djb2 해시(8자리 hex) 사용. 암호학적 해시가 아니며 IPv4 약 43억 개를 무차별 대입으로 역산 가능.

**수정 전:**
```typescript
export function hashIp(ip: string): string {
  let hash = 0;
  for (let i = 0; i < ip.length; i++) {
    const char = ip.charCodeAt(i);
    hash = ((hash << 5) - hash) + char;
    hash = hash & hash;
  }
  return Math.abs(hash).toString(16).padStart(8, "0");
}
```

**수정 후:**
```typescript
// 서버 전용: IP 익명화 (SHA-256). 클라이언트에서 호출 금지.
export function hashIp(ip: string): string {
  const salt = process.env.IP_HASH_SALT ?? "";
  // eslint-disable-next-line @typescript-eslint/no-require-imports
  const { createHash } = require("crypto") as typeof import("crypto");
  return createHash("sha256").update(ip + salt).digest("hex").slice(0, 16);
}
```

**설계 결정:**
- `import { createHash } from "crypto"` 모듈레벨 import는 클라이언트 번들 오염 위험 → `require("crypto")` 인라인 사용
- `utils.ts`는 클라이언트 컴포넌트에서도 import되는 파일이므로 모듈레벨 Node.js import 금지
- `IP_HASH_SALT` 환경변수로 레인보우 테이블 공격 방어
- 16자리 hex 출력 (SHA-256 전체 64자리 대신 저장 효율화)

#### [#7] 견적서 발송 — HTML form PATCH → Client Component fetch
**파일:** `src/app/dashboard/quotes/[id]/page.tsx`, **신규** `src/app/dashboard/quotes/[id]/SendButton.tsx`

**문제:** HTML `<form method="PATCH">`는 GET/POST만 지원. 발송 버튼이 실제로 동작하지 않았음.

**수정:** Server Component에 Client Component `SendButton.tsx` 분리 주입

```typescript
// SendButton.tsx — "use client"
const handleSend = async () => {
  const res = await fetch(`/api/quotes/${quoteId}`, {
    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ action: "send" }),
  });
  if (res.ok) router.refresh();
};
```

**설계 결정:** 페이지 전체를 클라이언트 컴포넌트로 전환하면 prisma 직접 호출 불가. 발송 버튼만 별도 클라이언트 컴포넌트로 분리하여 Server Component 구조 유지.

#### [#3] cn() — clsx 실사용
**파일:** `src/lib/utils.ts`

**문제:** `clsx`를 import하면서 실제로 사용하지 않고 `filter(Boolean).join(" ")`으로 대체. 객체/배열 형태의 조건부 클래스가 동작하지 않음.

```typescript
// 수정 전
import { type ClassValue } from "clsx";
export function cn(...inputs: ClassValue[]) {
  return inputs.filter(Boolean).join(" ");
}

// 수정 후
import { clsx, type ClassValue } from "clsx";
export function cn(...inputs: ClassValue[]) {
  return clsx(inputs);
}
```

> **참고:** `tailwind-merge`는 package.json에 없어 추가하지 않음. 향후 Tailwind 클래스 충돌이 발생하면 `pnpm add tailwind-merge` 후 `twMerge(clsx(inputs))`로 교체 필요.

#### [#8] requireUser() 에러 처리 구조 개선
**파일:** `intake/route.ts`, `pricing/route.ts`, `quotes/route.ts`, `quotes/[id]/route.ts`, `users/me/route.ts`

**문제:** `.catch(() => null)` 패턴으로 인증 에러와 DB/네트워크 에러를 구분 불가. DB 장애도 401 응답.

**수정:** try/catch 구조로 변경 (5개 라우트 전체)

```typescript
// 수정 전 (안티패턴)
const user = await requireUser().catch(() => null);
if (!user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

// 수정 후
let user: Awaited<ReturnType<typeof requireUser>>;
try { user = await requireUser(); } catch {
  return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
}
```
