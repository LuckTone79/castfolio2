## 2. 구현된 기능: `/create` 페이지

### 4단계 위저드 플로우

**Step 1 — 정보 입력**
- 방송인 이름 (한국어/영문)
- 포지션 선택 (아나운서, 쇼호스트, MC, 리포터 등 8가지)
- 다크 테마 UI, 스텝 인디케이터 포함

**Step 2 — 테마 선택**
- 7가지 전문 테마 갤러리 (기존 ALL_THEMES 재사용)
- 입력한 이름으로 실시간 테마 미리보기
- 선택 상태 시각적 피드백 (ring, scale)

**Step 3 — 페이지 편집 (빌더)**
- 기존 대시보드 빌더와 동일한 3패널 레이아웃:
  - 좌측: 섹션 네비게이션 (히어로, 프로필, 경력, 포트폴리오, 강점, 연락처)
  - 중앙: 실시간 미리보기 (데스크톱/모바일 전환)
  - 우측: 섹션별 편집 패널
- 테마 변경, 강조색 커스터마이징
- 다국어 편집 (KO/EN/ZH 전환)
- 섹션 활성화/비활성화 토글
- Tiptap WYSIWYG 리치 텍스트 편집기
- 로컬 이미지 업로드 (blob URL 기반)

**Step 4 — 미리보기**
- 완성된 PR 페이지 전체 미리보기
- 데스크톱/모바일 전환
- 다국어 전환
- 편집으로 돌아가기 기능

### 기술적 특징

- **서버 독립적**: 모든 데이터가 클라이언트 사이드에서 관리됨
- **이미지 처리**: `URL.createObjectURL()`을 사용한 blob URL로 서버 업로드 없이 즉시 미리보기
- **상태 관리**: React useState로 순수 로컬 상태 관리 (Zustand 스토어 구조 참고)
- **기존 컴포넌트 재사용**: PRPageRenderer, RichTextEditor, 테마 시스템 그대로 활용

## 3. 로그인 기능 숨김 처리

### 변경된 링크 매핑

| 페이지 | 변경 전 | 변경 후 |
|--------|---------|---------|
| 랜딩 헤더 "시작하기" | `/login` | `/create` |
| 랜딩 히어로 "무료로 시작하기" | `/login` | `/create` |
| 랜딩 가격 "지금 시작하기" | `/login` | `/create` |
| 랜딩 CTA "무료로 시작하기" | `/login` | `/create` |
| 랜딩 푸터 "로그인" | `/login` | `/create` ("시작하기"로 텍스트 변경) |
| 데모 헤더 "로그인" | `/login` | `/create` ("만들기"로 텍스트 변경) |
| 데모 테마 "시작하기" | `/login` | `/create` ("만들기"로 텍스트 변경) |
| 가이드 헤더 "로그인" | `/login` | `/create` ("만들기"로 텍스트 변경) |
| 가이드 CTA "무료로 시작하기" | `/login` | `/create` |

### 로그인 페이지 상태
- `/login` 경로는 존재하지만 어디서도 링크되지 않음 (숨김 처리)
- 향후 필요 시 즉시 복원 가능

## 6. 향후 개선 사항 (MVP 이후)

1. **서버 저장 연동**: 로그인 후 작성한 페이지를 서버에 저장/배포
2. **localStorage 임시 저장**: 브라우저 닫아도 작업 내용 유지
3. **PDF 내보내기**: 완성된 PR 페이지를 PDF로 다운로드
4. **회원가입 유도**: 미리보기 단계에서 "배포하려면 가입" CTA 강화
5. **로그인 기능 복원**: MVP 이후 인증 플로우 재활성화
