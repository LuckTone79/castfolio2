# Castfolio UI Architecture v2.0

> 전체 서비스 UI 구조 설계서
> 작성일: 2026-03-19

---

## 1. 사이트맵

```
castfolio.com/
│
├── /                           ← 랜딩페이지
├── /guide                      ← 이용 가이드
├── /demo                       ← 테마 갤러리 & 미리보기
├── /login                      ← 로그인/회원가입
│
├── /p/[slug]                   ← 🌐 퍼블릭 PR 페이지 (방송인 공개 프로필)
│
├── /public/                    ← 🔗 토큰 기반 공개 페이지
│   ├── intake/[token]          ← 자료 제출 폼
│   ├── quote/[token]           ← 견적서 확인
│   ├── review/[token]          ← 제출 내역 리뷰
│   └── delivered/[token]       ← 납품 확인
│
├── /dashboard/                 ← 👤 사용자 대시보드
│   ├── (overview)              ← 대시보드 홈
│   ├── talents/                ← 탤런트 관리
│   │   └── [id]/               ← 탤런트 상세
│   ├── projects/               ← 프로젝트 목록
│   │   └── [id]/               ← 프로젝트 상세
│   ├── builder/[projectId]     ← PR 페이지 빌더 (WYSIWYG)
│   ├── quotes/                 ← 견적서 목록
│   │   └── new                 ← 견적서 작성
│   ├── orders/                 ← 주문 목록
│   │   └── [id]/               ← 주문 상세
│   ├── pricing/                ← 상품 패키지 관리
│   │   ├── new                 ← 패키지 생성
│   │   └── [id]/               ← 패키지 편집
│   └── settings/               ← 계정 설정
│
└── /admin/                     ← 🔒 관리자
    ├── (overview)              ← 관리자 대시보드
    ├── orders/                 ← 전체 주문 모니터링
    ├── settlements/            ← 정산 관리
    ├── users/                  ← 사용자 관리
    ├── notifications/          ← 알림 발송
    ├── audit/                  ← 감사 로그
    └── system/                 ← 시스템 설정
```

## 2. 페이지 분류

### A. 외부 공용 페이지 (Public Marketing)

| # | 경로 | 목적 | 대상 사용자 | 핵심 요소 |
|---|------|------|------------|----------|
| 1 | `/` | 서비스 소개, 신규 가입 유도 | 미가입자, 검색 유입 | Hero, 기능 소개, 제작 과정, 가격표, FAQ, CTA |
| 2 | `/guide` | 서비스 이용 방법 안내 | 가입 고려자 | 5단계 가이드, 스크린샷, 영상 |
| 3 | `/demo` | 테마 갤러리, 실제 결과물 미리보기 | 가입 고려자 | 7개 테마 카드, 테마별 미리보기 모달 |
| 4 | `/login` | 로그인/회원가입 | 미인증 사용자 | Supabase Auth UI, 소셜 로그인 |

### B. 퍼블릭 PR 페이지 (Published Pages)

| # | 경로 | 목적 | 대상 사용자 | 핵심 요소 |
|---|------|------|------------|----------|
| 5 | `/p/[slug]` | 방송인 공개 PR 프로필 | 일반 대중, 캐스팅 디렉터 | Hero, Profile, Career, Portfolio, Strength, Contact, Footer |

### C. 토큰 기반 공개 페이지 (Token Public Flow)

| # | 경로 | 목적 | 대상 사용자 | 핵심 요소 |
|---|------|------|------------|----------|
| 6 | `/public/intake/[token]` | 방송인 자료 제출 | 탤런트(방송인) | 폼 필드, 파일 업로드, 진행 바 |
| 7 | `/public/quote/[token]` | 견적서 열람/승인 | 탤런트/클라이언트 | 금액 테이블, 승인/거절 버튼 |

| 9 | `/public/delivered/[token]` | 납품 확인 | 탤런트/클라이언트 | 완성 페이지 링크, QR 코드 |

## 6. 우선순위 (구현 순서)

### Phase 1: Core MVP (즉시 매출 가능한 최소 단위)

| 순위 | 페이지 | 이유 |
|------|--------|------|
| 1-1 | `/login` | 인증 없이 아무것도 못함 |
| 1-2 | `/dashboard` | 로그인 후 첫 화면 |
| 1-3 | `/dashboard/talents` + `[id]` | 탤런트 등록이 모든 흐름의 시작 |
| 1-4 | `/dashboard/projects` + `[id]` | 프로젝트 생성 → 제작 시작 |
| 1-5 | `/dashboard/builder/[projectId]` | 핵심 제품 (PR 페이지 제작) |
| 1-6 | `/p/[slug]` | 최종 결과물 (퍼블릭 PR 페이지) |

**Phase 1 완료 = "탤런트 등록 → 페이지 제작 → 발행" 가능**

### Phase 2: Commerce (매출 발생)

| 순위 | 페이지 | 이유 |
|------|--------|------|
| 2-1 | `/dashboard/pricing` + `new` + `[id]` | 상품 패키지 정의 |
| 2-2 | `/dashboard/quotes` + `new` | 견적서 발송 |
| 2-3 | `/public/quote/[token]` | 견적서 열람 |
| 2-4 | `/dashboard/orders` + `[id]` | 주문/결제 관리 |
| 2-5 | `/public/delivered/[token]` | 납품 확인 |

**Phase 2 완료 = "견적 → 결제 → 납품" 매출 흐름 가능**

### Phase 3: Automation (효율화)

| 순위 | 페이지 | 이유 |
|------|--------|------|
| 3-1 | `/public/intake/[token]` | 자료수집 자동화 |
| 3-2 | `/public/review/[token]` | 검수 프로세스 |
| 3-3 | `/dashboard/settings` | 브랜드 설정, 계정 관리 |

**Phase 3 완료 = 자료수집 ~ 검수까지 자동화**

### Phase 4: Growth (마케팅)

| 순위 | 페이지 | 이유 |
|------|--------|------|
| 4-1 | `/` | 랜딩페이지 (신규 유입) |
| 4-2 | `/guide` | 이용 가이드 |
| 4-3 | `/demo` | 테마 갤러리 |

**Phase 4 완료 = 마케팅 퍼널 완성**

### Phase 5: Admin (운영)

| 순위 | 페이지 | 이유 |
|------|--------|------|
| 5-1 | `/admin` | 관리자 대시보드 |
| 5-2 | `/admin/orders` | 주문 모니터링 |
| 5-3 | `/admin/settlements` | 월간 정산 |
| 5-4 | `/admin/users` | 사용자 관리 |
| 5-5 | `/admin/audit` + `notifications` + `system` | 운영 도구 |

**Phase 5 완료 = 플랫폼 운영 체제 완성**
